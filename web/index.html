⚠ Security Warning: The generated .cmd and .sh files will contain your IBM Storage Protect admin password in plain text so that dsmadmc can run unattended.
  • Run scripts only on trusted, secured hosts with the IBM Storage Protect administrative client installed.
  • Delete generated script files and output folder immediately after uploading results into this app.
  • Do not email, share, or store generated scripts in unsecured locations.
  • Consider resetting the admin password after the collection is complete.
  • On Windows, command-line passwords may be visible to local process inspection tools (Task Manager, Sysinternals).
  • On Unix/Linux, command-line arguments may be visible via tools such as ps while a query is running.

📦 Complete Collection

Generate one unified IBM Storage Protect collection containing configuration, schema discovery, retention, replication, and operational health queries. Both script formats run the same canonical query set and produce one complete XLSX workbook after import.

📝 Usage Notes

  • Windows .cmd and Unix/Linux .sh files run the same complete collection. Use whichever platform matches the installed administrative client.
  • Run .cmd from Windows Command Prompt (not PowerShell) on a host with dsmadmc.exe.
  • Run .sh from a Unix/Linux shell on a host with the administrative client installed, and use chmod +x <script>.sh if needed.
  • Each query writes one CSV file to the configured output directory. Errors are captured to collection_errors.log and echoed in real time.
  • Optional feature views can warn on older servers, but the confirmed schema-backed retention and storage-rule queries target exact table names for this server family.
  • If your password contains the characters ! ^ & | < > %, see the README for CMD escaping guidance.
  • The CSV output format uses comma-delimited values with a column-header row on line 1.
  • IBM documentation reference for server utilities and admin client options: Server commands, options, and utilities.

📊 Generate Complete XLSX Report

The report workbook contains the complete collection in one XLSX file with multiple worksheets organised by section. Sections with no data are shown as "No rows returned". Missing files are flagged as "Not imported".

Self-contained report download. XLSX files are generated locally in your browser with no CDN, no external script, and no network access required.

📑 Workbook Contents

Core Worksheets

  • Cover — report metadata and executive title page
  • Index — unified import status and row counts for every query
  • Server — status, database, log, options, backup health
  • Admins — administrator accounts
  • Nodes — node detail plus stale/locked/replication-health checks
  • Filespaces — filespace inventory
  • Policy — domains, policy sets, management classes, copy groups
  • Schedules — definitions plus failed/missed schedule health
  • Storage — pools, devices, libraries, paths, and storage-health checks
  • Volumes — disk, tape, library volumes, and volume history
  • Occupancy — occupancy and filespace usage summaries

Advanced Worksheets

  • Replication — defined servers, replication servers, storage/replication rules, protection stats
  • DR — DRM status and media
  • Retention — retention sets, retention rules, holds, and retention health
  • Advanced — containers and container-health queries
  • Activity/Health — activity log, summaries, copy failures, processes, sessions
  • Scripts — server-defined scripts
  • Schema — schema discovery for key IBM Storage Protect views
